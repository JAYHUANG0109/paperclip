# 四季藝術 AI 代理平台（Paperclip）— 整體規劃與進度報告

> 用途：內部會議簡報與未來工作參考。涵蓋整體架構（Paperclip 平台 + Google Chat 外掛 + Odoo API）、能力、限制、現況進度、未來規劃與里程碑、以及風險治理原則。

---

## 一、現況與痛點

目前四季各部門各自使用 Claude 企業帳號（一部門一帳號）。實務上遇到的問題：

- **分散、難管理**：帳號彼此獨立，沒有集中的管理介面，無法統一控管。
- **無法即時監看與協助**：管理者看不到同仁如何使用、遇到問題無法當場介入指導。
- **無個人化權限**：同部門共用一個帳號，無法依角色限定「誰能看、誰能做什麼」。
- **無稽核軌跡**：誰問了什麼、做了什麼，難以追蹤與檢討。
- **成本不透明**：共用帳號下，各部門 / 個人的 token 用量難以掌握。

## 二、願景

為**每一位員工**配置一個**專屬 AI 代理（agent）**：

- 每人用自己的 **@seasonart 帳號**，透過 **Google Chat** 直接和自己的代理對話（免安裝、免學新工具）。
- 每個代理依**角色**賦予對應**權限**——例如「園長」的代理擁有園長層級的存取，「財務專員」的代理只能接觸財務相關資料與工具。
- 具平台權限的**管理者**可在 Paperclip 後台**管理與監看**同仁代理的使用情形（存取範圍、可見層級、權限分級須經討論後明確定義，以降低風險）。
- 全公司統一在**一個平台、一套 UI** 上管理，取代目前分散的企業帳號。

## 三、整體架構與運作流程

三層串接：

1. **Paperclip 平台（核心大腦 ＋ 管理 UI）**：建立、設定、管理所有代理；定義角色與權限；提供後台監看與稽核。
2. **Google Chat 外掛（互動介面）**：員工在 Google Chat 以自己的帳號即可與專屬代理對話。
3. **Odoo 串接（API）**：代理透過 API 連到公司 Odoo（網上學習、採購、簽核、報表等）；採用**各帳號自己的 API 金鑰**，代理能接觸的資料＝該帳號在 Odoo 既有的權限（權限天然隨帳號綁定）。

**運作流程**：
員工 → 於 Google Chat 提問 → 專屬代理（Paperclip，依角色限定權限）→ 視需要呼叫工具 / 查詢 Odoo → 回覆員工；全程紀錄於平台供管理與稽核。

## 四、目前進度（已完成）

- ✅ Paperclip 平台已建置並上線。
- ✅ 已有 **3 個代理**實際運作中。
- ✅ 已透過自建的 **Google Chat 外掛**完成串接，員工可在 Chat 直接對話。
- ✅ 已打通 **Odoo API 連線**（以個別帳號金鑰存取，權限隨帳號）。
- ✅ 已建立可重複使用、可交付他人的**「Google Drive → Odoo 網上學習」匯入技能**（自動將雲端課程影片 / 簡報上架）。

## 五、目前能力

- 員工用 Google Chat 即可與代理互動（零學習成本）。
- 代理可依設定存取公司資料與工具、協助日常工作。
- Odoo 整合：可依帳號權限讀取 / 建立資料（如課程上架）。
- **可擴充的「技能 / 工具」機制**：需要新功能就新增一個技能，全平台共用、人人可自助。

## 六、限制與待解決（接下來要做）

1. **即時監看**（對標 Joanna 學姐以 Telegram 管理的模式）：讓管理者能**即時**看到同仁與代理的對話，必要時當場協助或介入。
2. **使用規範：僅限公務**——代理會消耗 token（成本），且基於隱私無須窺看私人事務；明確規範「**公務用代理、私事用個人聊天機器人**」。
3. **監管 / 守門代理**：設置一個監看整個平台與所有代理的代理，在人工檢視前先行**篩查提示與訊息（screening）**。
4. **權限越界風險**：代理可能不慎繞過設定的限制；亦可能有同仁嘗試**越權**存取 / 修改其無權接觸的內容——需主動防堵與告警。
5. **平台主控代理（master agent）**：建立一個全平台層級、**僅由最高權限者（創辦人）**可控制的總管代理。
6. **權限治理**：存取範圍、可見層級、權限分級須經討論明確定義。

## 七、Paperclip vs 現行 Claude 企業帳號

| 面向 | Paperclip 平台 | 現行 Claude 企業帳號 |
|---|---|---|
| 管理 | 集中於單一平台 ＋ UI | 各部門帳號分散、各自為政 |
| 權限 | 可依角色細分到個人 | 共用帳號，無法區分 |
| 監看 | 可即時監看與稽核 | 幾乎無法 |
| 擴充 | 技能 / 工具全公司共用 | 各自摸索、不易複製 |
| 成本 | 可分人 / 分部門掌握 | 不透明 |
| 整合 | 直接串接 Odoo 等系統 | 需個別處理 |

**結論**：Paperclip 以「**集中、可治理、有介面**」的方式推動全公司 AI 化，並讓代理管理更輕鬆。

## 八、里程碑 / 檢查點（以階段成果為準，非時間表）

- **M1　基礎建置（已完成）**：平台上線、3 代理運作、Google Chat 串接、Odoo API 打通。
- **M2　權限與角色模型**：定義各角色（園長 / 財務專員 / …）對應的權限與可見層級；建立各角色的**代理範本**。
- **M3　即時監看與使用規範**：管理者即時看對話（Joanna 模式）；發布「僅限公務」規範與 token 成本觀念。
- **M4　監管 / 守門代理**：自動篩查全平台提示與訊息、偵測並告警越權 / 繞過行為，再交人工複核。
- **M5　主控代理**：建立創辦人專屬、全平台最高權限的總管代理。
- **M6　全面推行**：逐部門為每位員工配置專屬代理。
- **M7　深化整合**：擴充 Odoo 可寫入能力與更多技能，邁向同仁自助。

## 九、治理原則（貫穿全程）

- **最小權限**：每個代理只給角色所需的最低存取。
- **全程稽核**：對話與操作留痕、可追溯。
- **人機協作**：高風險操作需人工複核。
- **隱私界線**：公務歸代理、私事歸個人，既保護同仁隱私、也控管成本。
